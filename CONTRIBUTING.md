# Contributing

## Scope

Contributions should improve the public portability, clarity, and usability of the architecture skill pack.

## Good contributions

- clarifying when to use a skill
- removing environment-specific assumptions
- improving public docs and install guidance
- tightening verification criteria
- adding support files that make an existing skill more reusable

## Please avoid

- turning one skill into multiple tiny fragments without a strong reason
- adding user-specific paths, hosts, or private operational assumptions
- rewriting a skill's core method just to change tone or style
- claiming a host-specific pattern is universal without a portability note
