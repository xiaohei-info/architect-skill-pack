# Example: Runtime A vs Runtime B Technology Adoption Analysis

## Purpose

This file is a worked example showing how to structure a technology-adoption comparison.
It is included as an example artifact shape, not as a current product recommendation.

## Suggested sections

- decision context
- evidence gathering log
- capability matrix
- fit-to-problem assessment
- final adopt/adapt/borrow/reject judgment
- trade-offs
- risks and mitigations
- non-goals
- verification path
- references

## Example framing

Use a comparison like this when evaluating:
- agent runtimes
- workflow engines
- orchestration platforms
- database or storage options
- internal build vs open-source adoption

## What to preserve from the example shape

- evidence is gathered from multiple source classes
- features are evaluated against the real target problem, not in isolation
- the recommendation explicitly states the reuse category
- risks and non-goals are named, not implied
- verification continues after the decision document is written
